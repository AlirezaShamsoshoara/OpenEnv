Connecting

Loading Environments

Four ways to connect — from local development to HuggingFace Hub

🏠

Local Server (base_url)

Simplest

Connect to an already running server. Start it with uvicorn or openenv serve.

# Async (recommended)
async with HackEnv(base_url="http://localhost:8000") as env:
    obs = await env.reset()
    result = await env.step(action)

# Sync wrapper
with HackEnv(base_url="http://localhost:8000").sync() as env:
    obs = env.reset()
📦

Docker Image (from_docker_image)

Auto-start

Starts a container from a local Docker image. Auto-finds an available port, waits for health check, cleans up on close.

env = await HackEnv.from_docker_image("hack-env:latest")

async with env:
    obs = await env.reset()
    result = await env.step(action)
# Container auto-removed on exit
🤗

HuggingFace Hub (from_env)

Auto-pull

Pulls a Docker image from the HuggingFace registry, starts the container, and connects. One-liner for any published environment.

# Docker mode (default) - pulls from registry.hf.space
env = await HackEnv.from_env("openenv/hack-env")

# UV mode - clones the repo and runs locally
env = await HackEnv.from_env(
    "openenv/hack-env", use_docker=False
)
⚡

UV Provider (Local Dev)

No Docker

Runs the environment directly via uv run. No container overhead. Supports hot reload for development.

from openenv.containers.runtime import UVProvider

provider = UVProvider(
    project_path="./hack_env", reload=True
)
url = provider.start(port=8000)
env = HackEnv(base_url=url, provider=provider)
💡 All methods use the same client API. Once connected, reset(), step(), state(), close() work identically regardless of how you loaded the environment.
