Deploying Environments

📦

Local Docker

Build and run locally. Each container is isolated with its own filesystem and network.

# Build image
$ docker build -t hack-env:latest -f server/Dockerfile .

# Run container
$ docker run -p 8000:8000 hack-env

# Or use openenv CLI
$ openenv build
$ openenv serve hack_env

HuggingFace Spaces

Push to HuggingFace for free hosting. Gets a public URL with auto-scaling.

# Push to HF Spaces
$ openenv push

# Connect from client
env = HackEnv(base_url=
  "https://openenv-hack-env.hf.space"
)

# Or use from_env factory
env = await HackEnv.from_env(
  "openenv/hack-env")

☁

Cloud Sandboxes

Daytona or Docker Swarm for production-grade isolation and scaling.

# Daytona cloud sandbox
env = await HackEnv(
  provider=DaytonaProvider()
)

# Docker Swarm (replicated)
provider = DockerSwarmProvider()
url = provider.start_container(
  "hack-env:latest",
  replicas=4
)

Dockerfile — Standard Pattern
# Stage 1: Build
ARG BASE_IMAGE=ghcr.io/meta-pytorch/openenv-base:latest
FROM ${BASE_IMAGE} AS builder
WORKDIR /app
COPY . /app/env
RUN uv sync --frozen --no-editable
# Stage 2: Runtime
FROM ${BASE_IMAGE}
COPY --from=builder /app/env /app/env
HEALTHCHECK CMD python -c "..."
CMD ["uvicorn", "server.app:app",
     "--host", "0.0.0.0", "--port", "8000"]
