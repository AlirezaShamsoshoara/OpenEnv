Connecting

Loading Environments

Six ways to connect — from local dev to cloud-scale production

🏠

Local Server (base_url)

Simplest

Connect to a running server. Start it yourself with uvicorn or openenv serve.

# Async
async with HackEnv(base_url="http://localhost:8000") as env:
    obs = await env.reset()

# Sync wrapper
with HackEnv(base_url="http://localhost:8000").sync() as env:
    obs = env.reset()
📦

Docker Image (from_docker_image)

Auto-start

Starts a container from a local Docker image. Auto-finds port, waits for health check, cleans up on close.

env = await HackEnv.from_docker_image("hack-env:latest")
async with env:
    obs = await env.reset()
# Container auto-removed on exit
🤗

HuggingFace Hub (from_env)

Auto-pull

Pulls Docker image from HuggingFace registry, starts container, and connects. One-liner for any published env.

# Docker mode (default) - pulls from registry.hf.space
env = await HackEnv.from_env("openenv/hack-env")

# UV mode - clones repo and runs locally
env = await HackEnv.from_env("openenv/hack-env", use_docker=False)
⚡

UV Provider (Local Dev)

No Docker

Runs environment directly via uv run. No container overhead. Hot reload for development.

from openenv.containers.runtime import UVProvider

provider = UVProvider(project_path="./hack_env", reload=True)
url = provider.start(port=8000)
env = HackEnv(base_url=url, provider=provider)
🔨

Docker Swarm (Replicated)

Scaling

Creates replicated services with load balancing. Ideal for parallel batch RL training.

from openenv.containers.runtime import DockerSwarmProvider

provider = DockerSwarmProvider()
url = provider.start_container(
    "hack-env:latest", replicas=4,
    cpu_limit="2.0", memory_limit="4g"
)
☁

Daytona Cloud (Sandboxes)

Cloud

Fully isolated cloud sandboxes. Each environment in its own VM. Signed preview URLs.

from openenv.containers.runtime import DaytonaProvider

provider = DaytonaProvider(api_key="...")
url = provider.start_container("hack-env:latest")
env = HackEnv(base_url=url, provider=provider)
💡 All methods use the same client API. Once connected, reset(), step(), state(), close() work identically regardless of how you loaded the environment.
