Reward Design

The Rubric System

PyTorch nn.Module-inspired composable reward functions — rewards stay inside the environment

Custom Rubric 
from openenv.rubrics import (
    Rubric, Sequential, WeightedSum, Gate
)

class FlagRubric(Rubric):
    """Reward for finding CTF flags."""
    def forward(self, action, obs) -> float:
        return len(obs.flags_found) / 5.0

class EfficiencyRubric(Rubric):
    """Penalize too many steps."""
    def forward(self, action, obs) -> float:
        return max(0, 1.0 - self.step_count * 0.05)
Composing Rubrics 
# Like nn.Sequential / nn.ModuleList
rubric = WeightedSum(
    rubrics=[
        FlagRubric(),          # 70% weight
        EfficiencyRubric(),    # 30% weight
    ],
    weights=[0.7, 0.3]
)

# Gate: zero reward if score below threshold
gated = Gate(rubric, threshold=0.1)

# Sequential: fail-fast on zero
strict = Sequential(
    FormatCheckRubric(),   # Must pass first
    CorrectnessRubric(),   # Then check answer
)

# Attach to environment
env = HackEnvironment(rubric=gated)
💡 Analogy: Rubrics are to rewards what nn.Module is to neural networks

● Rubric (Base)

Implement forward(action, obs) -> float. Supports hooks, children, state_dict, serialization.

● Sequential

Run rubrics in order. Fail-fast: returns 0 immediately if any rubric scores 0.

● WeightedSum

Parallel async eval via asyncio.gather. Weighted combination of multiple rubrics.

● Gate

Returns 0 if score below threshold. Useful for minimum quality requirements.

● TrajectoryRubric

Delayed rewards. Accumulates trajectory, scores at episode end. Supports ExponentialDiscounting.

Also Available

RubricList — dynamic list container
RubricDict — named access for multi-task environments
register_forward_hook — logging, metrics
